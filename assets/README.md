# 资源目录说明

这个 `assets` 目录只保留**当前项目实际使用**的资源。

原始完整资源集已经迁移到同级目录：

- `assets_origin`

后续如果要引入新的资源，请遵循下面的规则：

1. 先从 `assets_origin` 中找到原始文件。
2. 再按本目录的分类方式，复制到合适的位置。
3. 如果一个对象由多个文件组成，请为它单独建立一个子目录。
4. 优先保持“按用途分类”，不要再回到大平铺结构。

## 当前分类

- `settings`
  - 配置文件，例如 `art.ini`、`rules.ini`
- `palettes`
  - 所有调色盘与 `vpl` 查表
  - `theater`：剧院相关 palette
  - `ui`：UI 与图标 palette
  - `voxel`：体素相关 `vpl`
- `buildings`
  - 当前使用的建筑资源
  - 多文件建筑按建筑 ID 建子目录
- `vehicles`
  - 当前使用的载具资源
  - 例如 `rhino_tank` 目录中包含车体、炮塔、炮管的 `vxl/hva`
- `icons`
  - 侧栏与建造用 cameo 图标
  - 目前按 `base / defense / infantry / vehicles` 分类
- `ui`
  - 当前使用的 UI 主题资源
- `infantry`
  - 预留给步兵资源
- `effects`
  - 预留给特效资源
- `terrain`
  - 预留给地形与地图资源

## 目前约定

- 建筑 `shp` 可以放在 `buildings` 目录树中的任意子目录下，运行时会递归查找文件名。
- cameo 图标可以放在 `icons/cameo` 下的任意分类子目录中，运行时也会递归查找。
- 载具体素资源当前按对象目录直连加载，因此像 `rhino_tank` 这种多文件对象，目录层级要保持稳定。

## 目标

我们希望新的 `assets` 始终满足这几个原则：

- 清晰：一眼能看懂资源属于什么类别
- 收敛：只保留当前项目真正用到的资源
- 可扩展：后面新增单位、建筑、特效时有统一落点
- 可维护：避免重复资源和大平铺目录
